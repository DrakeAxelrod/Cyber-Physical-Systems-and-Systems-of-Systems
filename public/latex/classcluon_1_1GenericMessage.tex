\doxysubsection{Detailed Description}
\mbox{\hyperlink{classcluon_1_1GenericMessage}{Generic\+Message}} is providing an abstraction level to work with concrete messages. Therefore, it is acting as both, a Visitor to turn concrete messages into Generic\+Messages or as Visitable to access the contained data. \mbox{\hyperlink{classcluon_1_1GenericMessage}{Generic\+Message}} would use C++\textquotesingle{}s std\+::any type; to allow C++14 compilers, we use the backport from \mbox{\hyperlink{classlinb_1_1any}{linb\+::any}}.

Creating a \mbox{\hyperlink{classcluon_1_1GenericMessage}{Generic\+Message}}\+: There are several ways to create a \mbox{\hyperlink{classcluon_1_1GenericMessage}{Generic\+Message}}. The first option is to provide a message specification in ODVD format as result from \mbox{\hyperlink{classcluon_1_1MessageParser}{Message\+Parser}}, from which a \mbox{\hyperlink{classcluon_1_1GenericMessage}{Generic\+Message}} is created. This instance can then be visited afterwards by, for instance, an instance of Proto\+Decoder to set the \mbox{\hyperlink{classcluon_1_1GenericMessage}{Generic\+Message}}\textquotesingle{}s actual values.

1) This example demonstrates how to process a given message specification to decode a Proto-\/encoded byte sequence (in proto\+Encoded\+Data). The message specification is given in message\+Specification that is parsed from \mbox{\hyperlink{classcluon_1_1MessageParser}{Message\+Parser}}. On success, it is tried to decode the Proto-\/encoded data into a Generic\+Messsage representing an instance of \char`\"{}\+My\+Message\char`\"{}.

Using a message specification that does not match the serialized Proto data might result in unexpected behavior.
